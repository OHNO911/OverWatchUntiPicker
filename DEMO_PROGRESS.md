# OW Counter Lab Demo版 進捗管理（2026-04-05）

## 目的
Demo版として「敵構成に対するカウンターヒーロー提案」を、**ローカルで安定して体験できる状態**まで仕上げる。

---

## 現在の到達点（実装済み）

### 1. コア体験（MVP）
- 敵ヒーロー選択（タンク/ダメージ/サポートタブ）
- 推奨カウンター候補のランキング表示
- 上位候補のメリット/デメリット表示
- 5人チーム構成（A/B/C）の提案

### 2. 状態管理 / 永続化
- `appState` を中心に状態一元化
- `localStorage` によるデータ・設定の保存/復元
- DB編集の Undo/Redo

### 3. データ運用
- **ScriptableObject相当のデータ管理を JSON 化**
  - `data/heroes.json` を正本データに変更
  - `data/heroes.js` は JSON ローダー化
- デフォルト復元（初期データへリセット）対応

### 4. 外部データ連携
- OverFast API: ヒーロー画像取得
- Blizzard統計ページ（r.jina.ai 経由）: マップ別勝率の取得
- 12時間キャッシュ

### 5. UI機能
- マップ選択 / マップ補正ウェイトスライダー
- ロール順序の入れ替え
- DBモーダルによる相性データ編集

---

## Demo公開に対する評価

### ✅ Demoとして十分
- 主機能（選択→提案→比較）が一連で動く
- 編集と復元のサイクルがある
- マップ補正の「差分効果」を体感できる

### ⚠️ Demo前に強化したい
1. **自動テスト不足**（回帰リスク）
2. **エラー通知が弱い**（外部取得失敗時のUX）
3. **説明可能性の不足**（なぜこの候補かの詳細内訳）
