# OverWatchUntiPicker

OWの敵構成に対するアンチピック候補を出すシンプルなフロントエンドアプリです。

## 画面を実際に確認する方法

### 1) ローカルプレビューを起動

```bash
./scripts/dev-preview.sh
```

- デフォルトURL: `http://127.0.0.1:4173/`
- 別ポートで起動したい場合:

```bash
./scripts/dev-preview.sh 8080
```

### 2) 動作の最低限チェック（HTTP配信 + 主要要素）

> 先にプレビューサーバーを起動してから実行してください。

```bash
./scripts/quick-check.sh
```

- `index.html` が `200` で返ること
- タイトル文字列（`OWアンチくん`）が含まれること
- 主要グリッド要素（`hero-grid`）が含まれること

### 3) 画面確認の観点（手動）

改良前後で最低限この4点は見てください。

1. **レイアウト崩れ**: PC幅・スマホ幅でカードや結果欄が潰れていないか
2. **選択操作**: ヒーロー選択/解除、リセット、ロールキュー切替が直感的か
3. **結果の納得感**: 上位候補の理由ラベル（強い/弱い）が理解しやすいか
4. **DB編集の安全性**: モーダル内編集→保存→Undo/Redoが破綻しないか

---

## 内容を改良するためにやるべきこと（考察）

### A. 「なぜこの候補か」を説明可能にする（最優先）

現状はスコアと短いラベル表示が中心なので、以下を追加すると納得感が上がります。

- スコアの内訳（構成相性 + 個別相性）を分解表示
- 上位3候補に「採用理由」を定型文で表示
- 苦手要素が多い候補には注意表示を出す

### B. データ編集の信頼性を上げる

ユーザーがDBを触る設計なので、壊れにくさが重要です。

- JSONエクスポート/インポート機能（バックアップ容易化）
- 編集差分のプレビュー（保存前確認）
- 入力バリデーション（値域・重複・不整合）

### C. UX改善（意思決定コスト削減）

- 推奨結果に「推しポイント」タグを追加（対ダイブ強い等）
- ヒーロー検索/フィルタ（ロール・アーキタイプ）
- 「よく使う敵構成」プリセット保存

### D. 品質改善（継続開発向け）

- 計算ロジック（`core/calc.js`）のユニットテスト追加
- 状態管理（`core/state.js`）の境界ケーステスト追加
- UI自動確認（将来的にPlaywright導入）
